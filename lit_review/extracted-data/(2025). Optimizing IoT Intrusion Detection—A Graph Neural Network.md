**Basic Information**
- Full Title: Optimizing IoT Intrusion Detection—A Graph Neural Network Approach with Attribute-Based Graph Construction
- Authors: Tien Ngo, Jiao Yin, Yong-Feng Ge, Hua Wang
- Year: 2025
- Venue (Journal / Conference): Information (journal, MDPI)

**Problem Definition**
- Problem addressed: IoT intrusion detection where existing graph-based methods rely on physical network connections which may not effectively capture node relationships; need for more meaningful graph construction.
- Task type (Intrusion Detection / Anomaly Detection / NIDS / etc.): NIDS (Network Intrusion Detection System), including binary classification (benign vs. malicious) and multi-class classification (specific attack categories).

**Model Information**
- Model Family: Graph Neural Network (GNN)
- Method Name: Top-K Similarity Graph Framework (TKSGF)
- Key Idea (2-3 concise sentences): Instead of using physical network topology, edges are constructed based on Top-K attribute similarity (cosine similarity) between node feature vectors. This yields more meaningful node relationships. GraphSAGE is then used for inductive node embedding and intrusion classification.
- Architecture details (if available): Graph construction using cosine similarity and Top-K neighbor selection (K value variable, directed or undirected edges). GraphSAGE with aggregation functions (mean, LSTM, max-pooling), ReLU activation, learnable weight matrix, cross-entropy loss, softmax classifier. Hyperparameters: hidden channels 32, learning rate 0.01, Adam optimizer.

**Dataset Information**
- Dataset(s) used: NF-BoT-IoT, NF-ToN-IoT (from Machine-Learning-Based NIDS benchmark)
- NSL-KDD used (Yes/No): No
- Feature type: Flow-based (NetFlow format, tabular features converted to graph nodes)

**Results**
- Best reported performance: Binary NF-BoT-IoT F1=0.985222 (Directed k=7 GraphSAGE); Binary NF-ToN-IoT F1≈1.0000 (Undirected k=7 GraphSAGE); Multi-class NF-BoT-IoT F1=0.840447 (Undirected k=10 GraphSAGE); Multi-class NF-ToN-IoT F1=0.6325 (proposed approach). Outperforms all compared ML/DL and graph-based methods in Table 2.
- Comparison with baselines: Proposed approach outperforms Decision Tree, Random Forest, Naïve Bayes, E-GraphSAGE, Self-supervised Learning, Graph Isomorphism Network on all tasks (Table 2, p. 11). GraphSAGE consistently outperforms GCN and GAT across configurations (Figures 2-5, Tables 4-7).

**Baselines**
- Models compared against: Decision Tree, Random Forest, Naïve Bayes, E-GraphSAGE, Self-supervised Learning, Graph Isomorphism Network (Table 2, p. 11); also GCN and GAT within GNN comparison (Tables 4-7, Figures 2-5).

**Analysis**
- Strengths: Attribute-based Top-K similarity graph construction more meaningful than physical-link-based methods; GraphSAGE provides scalability and inductive learning; outperforms traditional ML and existing graph approaches; extensive ablation study on similarity metrics, thresholds, and graph directionality.
- Limitations: Struggles with specific attacks: Theft (NF-BoT-IoT, recall 0.01), DoS, MITM, Ransomware, Scanning, XSS (NF-ToN-IoT, all zero detection). GraphSAGE does not incorporate edge features, causing confusion between DoS and DDoS (all DoS classified as DDoS). GAT struggles with large dynamic graphs (performance drop, high runtime).
- Real-time capability (Yes / No / Not Reported): Not Reported (though scalability of GraphSAGE mentioned as efficient for large dynamic graphs, no explicit real-time inference claim)
- Computational complexity (if mentioned): Not explicitly quantified, but notes: GAT faces challenges with large graphs due to complexity (p. 3); GraphSAGE designed to scale efficiently for large dynamic IoT graphs (p. 3); GAT processing time significantly longer for k=9,10 (p. 12). Average run times reported in tables (e.g., 5–1000+ seconds).
