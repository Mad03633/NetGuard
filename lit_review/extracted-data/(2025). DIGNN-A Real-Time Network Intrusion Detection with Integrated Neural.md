**Basic Information**
- Full Title: DIGNN-A: Real-Time Network Intrusion Detection with Integrated Neural Networks Based on Dynamic Graph
- Authors: Jizhao Liu and Minghao Guo
- Year: 2025
- Venue (Journal / Conference): CMC (Computers, Materials & Continua), Vol. 82, No. 1

**Problem Definition**
- Problem addressed: Existing intrusion detection methods often overlook changes in network topology over time and interactions between hosts, treating network traffic as static graphs and neglecting temporal correlations. This limits detection accuracy against sophisticated, evolving threats.
- Task type: Network Intrusion Detection / NIDS (Real-time)

**Model Information**
- Model Family: Hybrid (Graph Neural Network + Attention + Integrated GNN with GRU-like gating)
- Method Name: DIGNN-A (Dynamic Integrated Neural Network with Attention)
- Key Idea: Represent network traffic as dynamic graph snapshots, convert each snapshot to a line graph to transform edge classification into node classification, apply graph convolution with multi-head attention (TransformerConv) to capture spatial features, and use an integrated GNN (GCNII-GRU) to capture temporal evolution across snapshots.
- Architecture details: Three main modules: (1) dynamic line graph processing module (constructs snapshots and line graphs), (2) graph convolutional module (TransformerConv layers with self-attention), (3) integrated graph neural network module (GCNII-based GRU cells with residual connections). Output is passed to a fully connected layer with softmax for binary classification.

**Dataset Information**
- Dataset(s) used: UNSW-NB15, NF-ToN-IoT-v2
- NSL-KDD used: No
- Feature type: Graph-based (derived from flow-based features: IP addresses, ports, protocol, state, bytes, timestamps)

**Evaluation (exact values as reported in the paper)**
- Accuracy: 99.30% (UNSW-NB15), 99.96% (NF-ToN-IoT-v2)
- Precision: 0.9743 (UNSW-NB15), 0.9998 (NF-ToN-IoT-v2)
- Recall: 0.9700 (UNSW-NB15), 0.9997 (NF-ToN-IoT-v2)
- F1-score: 0.9722 (UNSW-NB15), 0.9997 (NF-ToN-IoT-v2)
- AUC: Not Reported
- Detection Rate: (same as Recall)
- False Positive Rate: 3.00% (UNSW-NB15 as FAR), 0.03% (NF-ToN-IoT-v2 as FAR)
- Other metrics: False Alarm Rate (FAR) – see above

**Results**
- Best reported performance: On NF-ToN-IoT-v2, the method achieves 99.96% accuracy, 99.98% precision, 0.03% FAR, 99.97% recall, and 99.97% F1-score.
- Comparison with baselines: Outperforms all compared models (CNN-BiLSTM, Stacked Random Forest, VLSTM, CTGNN, E-GraphSAGE, DLGNN, MLP, Random Forest, DNN) on both datasets in accuracy and F1-score, with significantly lower false alarm rate on NF-ToN-IoT-v2.

**Baselines**
- Models compared against: CNN-BiLSTM, Stacked Random Forest, VLSTM, CTGNN, E-GraphSAGE, DLGNN, MLP, Random Forest, DNN

**Analysis**
- Strengths: Dynamic graph modeling with line graphs converts edge classification to node classification, enabling flexible representation. Attention mechanism (TransformerConv) improves feature extraction. Integrated GNN (GCNII-GRU) captures spatiotemporal correlations, achieving high accuracy and very low false alarm rates.
- Limitations: Interpretability issue inherent in graph neural networks. Added computational complexity from attention mechanism. Trade-off between accuracy and complexity. Not yet adapted to unsupervised or semi-supervised learning.
- Real-time capability: Yes (stated in title, abstract, and Section 1)
- Computational complexity: Message passing complexity O(|V|·max(d_i)). Line graph reduces maximum degree. Processing times: UNSW-NB15 total 80.02s (avg 64.67ms per snapshot); NF-ToN-IoT-v2 total 567.11s (avg 65.02ms per snapshot).
