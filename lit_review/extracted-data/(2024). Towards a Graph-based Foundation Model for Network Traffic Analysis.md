**Basic Information**
- Full Title: Towards a Graph-based Foundation Model for Network Traffic Analysis
- Authors: Louis Van Langendonck, Ismael Castell-Uroz, Pere Barlet-Ros
- Year: 2024
- Venue (Journal / Conference): Proceedings of the 3rd GNNet Workshop: Graph Neural Networking Workshop (GNNet '24), ACM

**Problem Definition**
- Problem addressed: Developing a foundation model for network traffic analysis that can learn general network dynamics and adapt to downstream tasks (intrusion detection, traffic classification, botnet classification) with minimal fine-tuning.
- Task type: Intrusion Detection / Traffic Classification / Botnet Detection

**Model Information**
- Model Family: Graph Neural Network / Foundation Model
- Method Name: Not Reported (described as “graph-based foundation model” or “spatio-temporal graph neural network”)
- Key Idea (2-3 concise sentences): Represent network traffic as a dynamic spatio-temporal graph at the flow level, using a self-supervised link prediction pretraining task to capture spatial and temporal dynamics. The pretrained model is then fine-tuned for various downstream network tasks, enabling efficient transfer learning with limited labeled data.
- Architecture details: Line-graph representation with flow nodes and source/destination IP nodes; bidirectional edges between IPs and flows; temporal edges between recurring nodes across time windows. Spatio-temporal GNN based on GraphSAGE framework, heterogeneous and convolutional. Model size ≈ 732,000 parameters.

**Dataset Information**
- Dataset(s) used: UNSW-NB15, ToN-IoT, BoT-IoT, CTU13, UPC-COMNET14
- NSL-KDD used (Yes/No): No
- Feature type: Flow-based (Netflow variants with bidirectional flow features)

**Evaluation (exact values as reported in the paper)**
- Accuracy: Not reported as absolute value; reported as percentage loss relative to optimal model performance (see Figure 2)
- Precision: Not reported
- Recall: Not reported
- F1-score: Uses macro F1 and weighted F1; specific absolute values not reported
- AUC: Not reported
- Detection Rate: Not reported
- False Positive Rate: Not reported
- Other metrics: Average performance increase of 6.87% for fine-tuned models over training from scratch (abstract, Section 4.2)

**Results**
- Best reported performance: Fine-tuned models achieve an average performance increase of 6.87% compared to training from scratch across three downstream tasks (intrusion detection, traffic classification, botnet classification). The pretrained model approaches optimal performance faster than training from scratch for most metrics and datasets (Figure 2).
- Comparison with baselines: Compared only against training from scratch (no external baseline models); fine-tuned models consistently outperform scratch-trained models in few-shot settings.

**Baselines**
- Models compared against: Training from scratch (same GNN architecture without pretraining)

**Analysis**
- Strengths: Captures structural characteristics of network traffic via graphs; model is ~1% the size of transformer-based foundation models (≈732k parameters vs. 60–770 million); efficient few-shot learning with limited labeled data.
- Limitations: Small pretraining dataset (only two unlabeled datasets in different network settings); traffic classification task furthest from pretraining context showed less improvement; no attention mechanism yet; requires significant up-scaling for operational deployment.
- Real-time capability (Yes / No / Not Reported): Not Reported
- Computational complexity (if mentioned): Model size ~732,000 parameters; described as “compact” and “more efficient to train and deploy due to geometrical reuse of parameters” (Section 1, p. 2)
