**Basic Information**
- Full Title: Foundation Models for Cybersecurity. A Comprehensive Multi-Modal Evaluation of TabPFN
- Authors: P. García, J. de Curto, I. de Zarza, J.C. Cano, C.T. Calafate
- Year: 2025
- Venue (Journal / Conference): Not Reported

**Problem Definition**
- Problem addressed: Evaluating foundation models (TabPFN, TabICL, LLMs) against traditional machine learning for tabular intrusion detection, particularly addressing class imbalance and rare threat detection where traditional methods achieve high overall accuracy but fail on minority attack classes.
- Task type (Intrusion Detection / Anomaly Detection / NIDS / etc.): Intrusion Detection / NIDS

**Model Information**
- Model Family: Foundation Model
- Method Name: TabPFN (Tabular Prior-Data Fitted Network), TabICL (Tabular In-Context Learning), LLM (Gemini family)
- Key Idea (2-3 concise sentences): TabPFN approximates Bayesian inference without per-dataset training using a probabilistic transformer. TabICL uses a two-stage column-then-row attention mechanism for efficient in-context learning on large tabular data. LLMs convert tabular data to structured text for few-shot classification via prompting.
- Architecture details (if available): TabPFN: probabilistic transformer with 4 estimators, memory-saving mode, batch size 20k. TabICL: two-stage architecture (column-then-row attention), v1.1 checkpoint, 16 ensemble members, mixed precision, batch size 50k. LLM: Gemini 2.0 models with temperature 0.9, top_p 0.8, majority voting.

**Dataset Information**
- Dataset(s) used: CIC-IDS2017, N-BaIoT, CIC-UNSW
- NSL-KDD used (Yes/No): No
- Feature type: Tabular / Flow-based

**Evaluation (exact values as reported in the paper)**
- Accuracy: TabICL 0.9959 (Variant 1, CIC-IDS2017, Table 1, p.12); TabPFN 0.9896 (Variant 1); Random Forest 0.9947 (Variant 1)
- Precision: Per-class values reported; e.g., TabICL Heartbleed precision 1.0000 (Table 4, p.15)
- Recall: TabICL Heartbleed recall 1.0000; TabPFN Infiltration recall 0.9231 (Table 4, p.15)
- F1-score: TabICL Heartbleed F1 1.0000; TabICL Infiltration F1 0.2692 (Table 4, p.15)
- AUC: Not Reported
- Detection Rate: Not explicitly reported; recall serves as detection rate per class.
- False Positive Rate: Not Reported
- Other metrics: Per-class precision/recall/F1 for minority classes (Tables 4,5,6); training/testing latency (Tables 8-11, pp.16-17)

**Results**
- Best reported performance: TabICL achieves 99.59% accuracy on CIC-IDS2017 Variant 1 (p.12) and uniquely achieves non-zero recall across all 15 attack classes including Heartbleed and Infiltration (p.12, Fig.2). TabPFN also achieves non-zero recall across all classes.
- Comparison with baselines: Foundation models (TabPFN, TabICL) outperform traditional ensembles (Random Forest, k-NN) and TabNet on minority-class detection. Traditional methods achieve >99% overall accuracy but have zero recall on classes like Heartbleed, Infiltration, and Web Attack XSS (Table 4, p.15). TabICL shows consistent performance across datasets (CIC-IDS2017, N-BaIoT, CIC-UNSW) while traditional methods exhibit dataset-specific variations.

**Baselines**
- Models compared against: Logistic Regression, Support Vector Machine (SVM), k-Nearest Neighbors (k-NN), Decision Tree, Random Forest, TabNet (with/without SMOTE)

**Analysis**
- Strengths: Comprehensive detection of all attack classes including rare threats; no hyperparameter tuning or per-dataset retraining; robust generalization across diverse datasets (CIC-IDS2017, N-BaIoT, CIC-UNSW); superior minority-class recall.
- Limitations: High GPU memory requirements (TabPFN >20GB VRAM); TabPFN inference latency up to 97 minutes on full dataset; LLMs constrained by context size (45-100 samples) and API latency >1s per prediction; reduced interpretability compared to tree-based ensembles; potential adversarial vulnerability.
- Real-time capability (Yes / No / Not Reported): Not Reported (TabICL per-sample latency ~0.3-0.5 ms; TabPFN per-sample ~0.8-7.8 ms; but full-dataset inference takes minutes to hours, and paper notes real-time deployment requires careful consideration of inference latency)
- Computational complexity (if mentioned): TabPFN has quadratic scaling in number of in-context samples (p.17); TabICL more scalable with sub-second setup and minute-level inference on full datasets (pp.16-18); TabNet training takes 1-3+ hours.
