**Basic Information**
- Full Title: Transformer Tokenization Strategies for NIDS
- Authors: G. Aksholak, A. B., R. Magazov, K. Kaplan
- Year: 2026
- Venue (Journal / Conference): Computers (MDPI)

**Problem Definition**
- Problem addressed: Network intrusion detection challenges including lack of inherent ordering in network traffic features, extreme class imbalance (benign traffic >80%), and wide variation in reported accuracies (57-95%). The paper isolates and quantifies the impact of tokenization strategies on Transformer-based IDS performance.
- Task type: Intrusion Detection / NIDS

**Model Information**
- Model Family: Transformer
- Method Name: Feature-wise Tokenization Transformer (FTT), Sample-wise Transformer-Baseline (STB), Sample-wise Transformer-Optimized (STO)
- Key Idea: Tokenization choice alone accounts for a 37.43 percentage point accuracy gap. Sample-wise tokenization (treating entire feature vector as a single token) vastly outperforms feature-wise tokenization. Architectural refinements such as Batch Normalization and capped class weights further improve stability and minority-class detection under extreme imbalance.
- Architecture details: STO uses sample-wise tokenization, initial Dense(128) embedding with ReLU, Batch Normalization after embedding and classification head, 3 Transformer blocks with 8 attention heads, dense feed-forward layers (256→128), dropout rates 0.1-0.3, capped class weights (max=10.0).

**Dataset Information**
- Dataset(s) used: CICIDS2017
- NSL-KDD used: No
- Feature type: Tabular / Flow-based (78 numerical features extracted with CICFlowMeter)

**Evaluation (exact values as reported in the paper)**
- Accuracy: 94.52% (STO on 100K data) / 95.22% (STO on 150K data)
- Precision: 93.87%
- Recall: 93.21%
- F1-score: 93.54%
- AUC: micro-average AUC = 0.998, macro-average AUC = 0.980
- Detection Rate: Not Reported
- False Positive Rate: Not Reported
- Other metrics: Minority-class recall increase of 7-12% relative to baseline; Web Attack XSS (4 samples) achieves AUC 0.9811

**Results**
- Best reported performance: Accuracy 95.22% (with 150K samples), macro-average AUC 0.98, micro-average AUC 0.998. Tokenization transition (FTT→STB) yields +22.38 percentage points; architectural refinements (STB→STO) yield +15.05 points; data expansion (100K→150K) yields only +0.70 points.
- Comparison with baselines: Outperforms FTT (57.09% accuracy) and STB (79.47% accuracy). Compared to state-of-the-art methods (Random Forest 99.49%, DNN 96.0%, BERT-MLP 99.39%) but those use aggregated class settings (7-9 classes) while this preserves all 11 classes including ultra-rare attacks.

**Baselines**
- Models compared against: Random Forest, DNN, CNN-LSTM, CNN, Vision Transformer, TabNet-IDS, BERT-MLP, plus their own FTT and STB.

**Analysis**
- Strengths: Systematic isolation of tokenization impact (59.8% of total improvement); architectural stability mechanisms (BatchNorm, capped weights) effectively mitigate extreme class imbalance without synthetic data; achieves high macro-AUC (0.98) even for classes with as few as 4 samples.
- Limitations: Only evaluated on CICIDS2017 (no cross-dataset validation); offline evaluation only, no real-time or deployment-level performance analysis; minority classes with <14 samples remain challenging; absolute accuracy lower than classical tree-based methods on static tabular benchmarks.
- Real-time capability: No (offline evaluation; inference latency, throughput, and operational constraints considered beyond scope)
- Computational complexity: Not Reported
