**Basic Information**
- Full Title: Network Intrusion Detection with Edge-Directed Graph Multi-Head Attention Networks
- Authors: Xiang Li, Jing Zhang, Yali Yuan, Cangqi Zhou
- Year: 2023
- Venue (Journal / Conference): Not Reported (appears as "JOURNAL OF LATEX CLASS FILES, VOL. 99, NO. 10, OCTOBER 2023")

**Problem Definition**
- Problem addressed: Existing graph neural network (GNN) models for intrusion detection treat node mean aggregation equally, ignoring differences in correlations among nodes and edges, and do not account for edge features or directionality of network flows.
- Task type: Network Intrusion Detection System (NIDS), multi-class classification (distinguishing multiple attack types and benign traffic)

**Model Information**
- Model Family: Graph Neural Network (specifically Graph Attention Network with edge features and direction)
- Method Name: EDGMAT (Edge-Directed Graph Multi-Head Attention Networks)
- Key Idea (2-3 concise sentences): The model introduces a multi-head attention mechanism that aggregates nodes and edges with different weights, incorporating both edge features and the directed nature of network flows. By assigning higher weights to nodes and edges with high similarity, it better explores correlations and improves classification accuracy.
- Architecture details (if available): 4-layer GNN: two EDGMAT convolutional layers, ReLU activation, softmax output layer, dropout of 0.2. Uses cross-entropy loss with class weights, Adam optimizer (learning rate 0.01), training iterations 100–200. Residual connections added during node aggregation.

**Dataset Information**
- Dataset(s) used: BoT-IoT, ToN-IoT, NF-BoT-IoT, NF-ToN-IoT
- NSL-KDD used (Yes/No): No
- Feature type: Flow-based (NetFlow format for NF-* variants; original features include source/destination IP, source/destination port, packets, bytes, duration, protocol flags)

**Evaluation (exact values as reported in the paper)**
- Accuracy: Not Reported
- Precision: Weighted precision 95.88% (NF-BoT-IoT transductive, Table I), 96.11% (NF-BoT-IoT inductive, Table V)
- Recall: Weighted recall 88.79% (NF-BoT-IoT transductive, Table I), 89.41% (NF-BoT-IoT inductive, Table V)
- F1-score: Weighted F1 0.91 (NF-BoT-IoT transductive), 0.92 (NF-BoT-IoT inductive), 0.99 (BoT-IoT transductive/inductive), 0.74 (NF-ToN-IoT transductive), 0.67 (NF-ToN-IoT inductive), 0.91/0.92 (ToN-IoT)
- AUC: Not Reported
- Detection Rate: Not Reported (recall reported instead)
- False Positive Rate: Not Reported
- Other metrics: Per-class precision, recall, F1-score for all datasets (Tables I–VIII)

**Results**
- Best reported performance: Weighted F1-score of 0.99 on BoT-IoT dataset (both transductive and inductive settings) – Table II and Table VI.
- Comparison with baselines: EDGMAT outperforms KNN, XGBoost, Extra-Trees, and E-GraphSAGE on most datasets. On NF-BoT-IoT: EDGMAT W-F1=0.91 vs KNN 0.80, XGBoost 0.83, Extra-Trees 0.77. On NF-ToN-IoT: EDGMAT W-F1=0.74 vs KNN 0.61, XGBoost 0.60, Extra-Trees 0.60. On ToN-IoT: EDGMAT W-F1=0.91 vs XGBoost 0.88, Extra-Trees 0.87 (KNN 0.97 slightly higher). On BoT-IoT: EDGMAT W-F1=0.99 vs E-GraphSAGE 0.99 (tie).

**Baselines**
- Models compared against: E-GraphSAGE, K-Nearest Neighbors (KNN), XGBoost, Extra-Trees Classifier

**Analysis**
- Strengths: Incorporates edge features and directionality of network flows; uses multi-head attention for weighted aggregation; outperforms state-of-the-art on most datasets; works under both static (transductive) and dynamic (inductive) topologies.
- Limitations: Poor performance on minority classes (e.g., Theft, Injection, MIMT, Ransomware) especially in inductive setting; imbalanced datasets cause low F1 for rare categories; no evaluation of computational overhead or training time.
- Real-time capability (Yes / No / Not Reported): Not Reported
- Computational complexity (if mentioned): Not reported; however, the paper notes future work includes "Mitigating GPU memory usage and training duration" (Section VI, p. 11).
